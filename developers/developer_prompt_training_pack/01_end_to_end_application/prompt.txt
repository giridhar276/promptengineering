Build a complete runnable Customer Support Ticket Management application named SupportDesk.

STACK
- Python 3.11, FastAPI, SQLAlchemy 2.x, Alembic
- React 18
- PostgreSQL 15
- JWT authentication
- Docker Compose
- pytest

REQUIREMENTS
1. Roles: agent, manager, administrator.
2. Users log in using email and password.
3. Administrators create, update, deactivate, and list users.
4. Agents create tickets and update tickets assigned to them.
5. Managers assign and update any ticket.
6. Administrators manage all users and tickets.
7. Ticket fields: id, title, description, customer_email, priority, category, status, assignee_id, created_at, updated_at.
8. Priorities: LOW, MEDIUM, HIGH, CRITICAL.
9. Statuses: OPEN, IN_PROGRESS, RESOLVED, CLOSED.
10. Only managers and administrators can reopen CLOSED tickets.
11. Every ticket change creates an immutable audit record.
12. Search by ticket ID, customer email, status, priority, category, and assignee.
13. Add pagination, sorting, validation, structured logging, correlation IDs, centralized errors, health, readiness, and version endpoints.
14. Use secure password hashing and environment variables.
15. Add indexes for searchable fields.
